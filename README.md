# CS20-Software-Test-Automation-QA-2026
CS-320 Summery and Retrospective

How can I ensure that my code, program, or software is functional and secure?
-> We want to make sure that we follow every phase of the Software Development Lifecycle (SDLC).  it involves functional requirements in which we know what the system does, and it's non-funcational security requirments.  How it is conducted securely during the planning phase.  By maintaining secure coding standards like output encoding and input validation, strengtening principle of least privilege to limit access, and finally identifying attack vectors with threat modeling are all great examples on preventing vulnerabilities.

How do I interpret user needs and incorporate them into a program?
-> So we have to know the difference between functional requirements and non-functional requirements.  We also have to get user needs through stakeholder interviews and documentation, then translate them into architecture diagrams using pseudocode, diagram data flow models.  This will provide security controls, that supports specific functions that users/clients will be interested in but it won't suffer the integrity of usability.

How do I approach designing software?
-> The principles of "Secure by design" and "Secure by Default" will be the best approach.  Security from the beginning will be embedded.  Use mulitiple layers of protection like firewalls, access controls, and secure communication protocols for your defense in depth.  And fail-secure defaults so it wold deny access to the system if there are any errors. Finally, use separation of duties that no developer can both write and deploy code to production.  Thus, reducing the risk of malicious activity and critical errors.
